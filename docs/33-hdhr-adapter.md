# HDHomeRun Adapter

> Copyright (c) 2026 Perry Kivolowitz. All rights reserved.
> Licensed under the [MIT License](../LICENSE).

GlowUp integrates HDHomeRun network tuners via their HTTP API.  The
`HDHomeRunAdapter` polls tuner status every 10 seconds and caches the
channel lineup hourly.  Tuner signal quality is written to the
`SignalBus` for use by operators and dashboard tiles.

---

## Architecture

```
+-------------------+       +-------------------+
|  HDHomeRun        |  HTTP |  GlowUp Server    |
|  (hdhomerun.local)|<----->|  HDHomeRunAdapter  |
|  Port 5004        |       |  (PollingAdapterBase)
+-------------------+       +-------------------+
                                    |
                              SignalBus
                                    |
                         +----------+---------+
                         |                    |
                    /home tiles          Operators
                   (tuner status)       (signal alerts)
```

- **HDHomeRun** -- Network tuner accessible at `http://<ip>:5004`.
  Exposes channel lineup, tuner status, signal diagnostics, and
  MPEG-TS streams via a simple HTTP/JSON API.
- **HDHomeRunAdapter** -- Server-side adapter that polls the
  device and writes signal metrics to the `SignalBus`.

---

## Inheritance

`HDHomeRunAdapter` inherits from `PollingAdapterBase` (see [Chapter
27: Adapter Base Classes](27-adapter-base.md)).  The base class
provides:

- Daemon thread with configurable poll interval
- Interruptible sleep (5-second chunks) for responsive shutdown
- `_check_prerequisites()` hook for config validation

`HDHomeRunAdapter` implements `_do_poll()` to fetch tuner status and
optionally refresh the channel lineup.

---

## Configuration

Add an `hdhr` section to `server.json`:

```json
{
    "hdhr": {
        "host": "hdhomerun.local",
        "port": 5004,
        "poll_interval": 10
    }
}
```

- **host** -- Hostname or IP of the HDHomeRun device.  Default:
  `hdhomerun.local` (mDNS).
- **port** -- HTTP API port.  Default: `5004`.
- **poll_interval** -- Seconds between status polls.  Default: `10`.

---

## HDHomeRun HTTP API Endpoints

The adapter uses these endpoints:

- `/status.json` -- Device info and per-tuner status (channel,
  frequency, signal strength/quality/symbol quality).
- `/lineup.json` -- Available channels with names and numbers.
- `/auto/v{channel}` -- MPEG-TS live stream URL (not polled, but
  generated by `get_stream_url()` for dashboard use).

All requests timeout after 5 seconds.  Failures are logged at
`DEBUG` level and silently retried on the next poll cycle.

---

## Signal Bus Integration

For each active tuner (one that has a tuned channel), the adapter
writes two signals per poll:

- `hdhr:tuner{N}:signal_quality` -- Signal quality percentage
  (0--100).  Below 50 may cause dropouts.
- `hdhr:tuner{N}:symbol_quality` -- Symbol quality percentage.
  Should be 100 for error-free decoding.

These signals can be consumed by operators (e.g., alert when signal
drops below a threshold) or displayed on the `/home` dashboard.

---

## Dashboard API

The adapter exposes four methods for the dashboard handler:

- **`get_tuner_status()`** -- Returns a list of tuner dicts, each
  containing: `tuner` (index), `channel` (name), `channel_number`,
  `frequency`, `signal_strength`, `signal_quality`,
  `symbol_quality`, `active` (bool).

- **`get_device_info()`** -- Returns model, firmware version, device
  ID, tuner count, and base URL.

- **`get_lineup()`** -- Returns available channels: `number`, `name`,
  `hd` (bool).

- **`get_stream_url(channel)`** -- Returns the HTTP URL for a live
  MPEG-TS stream of the given virtual channel number.

---

## Signal Interpretation

| Metric | Good | Fair | Poor |
|--------|------|------|------|
| Signal Strength | >75% | 50--75% | <50% |
| Signal Quality | >75% | 50--75% | <50% |
| Symbol Quality | 100% | 90--99% | <90% |

- Symbol quality below 100% means visible glitches or dropouts.
- High signal strength with low quality indicates multipath
  interference.
- Low signal strength across all tuners indicates antenna or cable
  issues.

## Dependencies

None beyond Python stdlib.  The adapter uses `urllib.request` for
HTTP and `json` for parsing.
